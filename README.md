# dotfiles

Personal dotfiles

## Setup on a new machine

```sh
# 1. Install Homebrew (https://brew.sh), then:
git clone git@github.com:mchudoba/dotfiles.git ~/dev/dotfiles
cd ~/dev/dotfiles
./install.sh
```

`install.sh` symlinks the configs into place, backs up any pre-existing real
files as `<file>.bak`, seeds the local config templates, and offers to run
`brew bundle`. It's idempotent — safe to re-run.

## What's here

| File | Linked to | Purpose |
|------|-----------|---------|
| `zshrc` | `~/.zshrc` | zsh config (powerlevel10k prompt, plugins, aliases) |
| `p10k.zsh` | `~/.p10k.zsh` | powerlevel10k prompt config (generated by `p10k configure`) |
| `zshrc.local.example` | copied to `~/.zshrc.local` | machine-local / work config, **not** symlinked |
| `gitconfig` | `~/.config/git/config` | git settings |
| `gitconfig.local.example` | copied to `~/.config/git/config.local` | machine-local git config, **not** symlinked |
| `gitconfig.work.example` | copied to `~/.config/git/config.work` | work git config, **not** symlinked |
| `gitignore_global` | `~/.config/git/ignore` | global gitignore |
| `fdignore` | `~/.config/fd/ignore` | dirs `fd` skips outside a git repo (fzf walks with `fd`) |
| `vimrc` | `~/.vimrc` | vim config (no plugins, stock vim 9.1+) |
| `Brewfile` | — | `brew bundle` package list |

## Notes

- **Local config** (project aliases, tokens) lives in `~/.zshrc.local`, sourced at
  the end of `~/.zshrc`. Edit it freely; it never touches this repo.
- **Git config lives under `~/.config/git/`**, git's XDG location.
- **Git identity**: `gitconfig` commits the personal email and nothing else.
  `config.local` matches work repos by their remote org and pulls the work email
  in from `config.work`. Both are untracked, so no employer details land in this
  public repo. Verify with `git -C <repo> config user.email`.
- **Rebuilding the prompt config**: run `p10k configure`, then copy the result
  back over `p10k.zsh` (it's a real file here, symlinked into `$HOME`).
